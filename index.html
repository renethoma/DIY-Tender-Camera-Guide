Two complete approaches to building a modular digital camera from a single-board computer, open-source software, and off-the-shelf parts.
Built for repairability, transparency, and longevity.

About this project

One goal,
two paths.

Both builds share the same permacomputing philosophy — every component is replaceable, every dependency is open-source, and the entire process is documented so anyone can replicate it.

What they share

OS base Ubuntu 24.04 Ubuntu 22.04
Camera API V4L2 / UVC V4L2 / MIPI
Display Waveshare 4.3" Waveshare 3.5"
App layer Python + Kivy Python + OpenCV
Driver work Minimal Significant

Which should you build?

Sweet Potato — if you want the simplest path. USB peripherals mean zero driver configuration. Great for beginners.

Orange Pi 5 — if you want more power and resolution and don't mind device tree overlays and compiling a custom touch driver.

Hardware

Libre Computer AML-S905X-CC-V2
Quad-core A53 @ 1.5GHz · 2GB DDR3 · 4× USB 2.0 · 100Mbps Ethernet
Sony IMX179 · 8MP USB Camera
UVC-compliant — zero driver work. Plug in and it works as /dev/video1.
Waveshare 4.3" HDMI LCD
Capacitive USB touch — also plug & play. Appears as /dev/input/event0.

Setup steps

1

Flash Armbian 25.11 to MicroSD

Ubuntu 24.04 base. Boot, login as mycamera.

2

Verify camera on /dev/video1

video0 is the SoC decoder — always use video1 for the camera.

v4l2-ctl --list-devices
3

Install packages

sudo apt install fswebcam evtest fbi
4

Capture your first photo

fswebcam -d /dev/video1 -r 3264x2448 \ --no-banner photo.jpg
5

Verify touch input

Select device 0 — touch the screen to see ABS_X / ABS_Y events.

sudo evtest
6

Install Kivy UI framework

Touch-native Python UI. Renders to framebuffer — no X11 needed.

sudo apt install python3-kivy
⚠ FHNW Network note: ping is blocked by the university firewall. Use sudo apt update to test connectivity instead.

Hardware

Orange Pi 5 · RK3588S
4+4 core A76/A55 · 4–8GB LPDDR4X · Gigabit Ethernet · PCIe M.2
OV13850 · 13MP MIPI Camera
Requires Device Tree overlay. Connects to CAM1 via FPC ribbon cable.
Waveshare 3.5" HDMI LCD — Resistive touch
XPT2046 SPI controller. Kernel driver disabled — needs custom daemon.

Setup steps

1

Flash Ubuntu 22.04 XFCE image

Official Orange Pi 5 image. Boot, login as orangepi.

2

Enable Device Tree overlays

Enables the MIPI camera and SPI4 bus. Use m0, not m1 — m1 breaks ethernet.

overlays=ov13850-c1 spi4-m0-cs1-spidev
3

Verify camera on /dev/video11

dmesg | grep ov13850 v4l2-ctl --list-devices
4

Build touch driver (3 patches needed)

WiringOP for GPIO + ADS7846 daemon patched for software CS on pin 26.

sudo apt install libx11-dev libxtst-dev git clone .../WiringOP && ./build git clone .../ADS7846-X11-daemon # apply 3 patches → make → calibrate
5

Install Python dependencies

sudo apt install python3-opencv
6

Run the camera app

Full source code included in the guide. P=photo V=video S=settings Q=quit.

python3 camera_app.py /dev/video11
⚠ FPC cable warning: The ribbon cable connecting the OV13850 is fragile and the most common failure point. If the camera isn't detected, replace the cable first.
